# OpenDinq v0.9-alpha

v0.9-alpha adds the Quality Layer for the product alpha.

## What Changed

- Added a deterministic claim quality pipeline.
- Improved card generation for summaries, skills, works, research, timelines, and manual notes.
- Added generated card quality metadata in `dataJson`.
- Added optional evidence-constrained LLM card rewrite behind env flags.
- Added Discover score breakdowns and stronger rule/full-text ranking.
- Improved demo profiles for AI agent builders, product designers, ML researchers, and open-source infrastructure engineers.

## Claim Quality

Claims are normalized, deduped, ranked, and quality-scored before card generation. Duplicate claims merge evidence refs and prefer higher confidence. Rejected claims are excluded from public and Discover ranking.

## Card Quality

Cards remain deterministic by default. Summary and skill cards use approved/high-quality claims, Works cards rank stronger artifacts first, Research cards require research evidence, and Timeline cards require dates.

## Discover Quality

Search results include `scoreBreakdown` with claim, card, artifact, skill, evidence, publish boost, recency, and final scores. Exact phrase and evidence-backed matches now rank above weak profile-text matches.

## Optional LLM Rewrite

Disabled by default. Enable with:

```bash
OPEN_DINQ_ENABLE_LLM_REWRITE=true
OPENAI_API_KEY=...
```

The rewrite is evidence-constrained and falls back to deterministic cards if validation fails.

## Limitations

- No auth or ownership.
- No teams or billing.
- No production semantic/vector search.
- LLM rewrite is optional and experimental.
- Card quality depends on source evidence.
- GitHub token is recommended for GitHub import.
- No LinkedIn/X scraping, browser automation, private DINQ APIs, or copied DINQ branding/assets.
