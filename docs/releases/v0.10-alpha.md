# OpenDinq v0.10-alpha

v0.10-alpha makes profile generation AI-first while preserving deterministic local mode.

## What Changed

- Added single-input LLM-powered generation on `/generate`.
- Added `POST /api/profiles/plan` for non-persistent plan previews.
- Added `POST /api/profiles/generate-ai` for plan-plus-generate flow.
- Added `@opendinq/llm` intent planning and evidence-backed claim synthesis.
- Added MCP tools `opendinq_plan_profile_generation` and `opendinq_generate_profile_ai`.
- Moved explicit source fields under collapsed Advanced sources.

## Env Vars

```bash
OPEN_DINQ_ENABLE_LLM_GENERATION=true
OPEN_DINQ_LLM_PROVIDER=openai-compatible
OPEN_DINQ_LLM_MODEL=gpt-4.1-mini
OPEN_DINQ_LLM_API_KEY=...
OPEN_DINQ_LLM_BASE_URL=https://api.openai.com/v1 # optional
```

Card rewrite remains separately gated:

```bash
OPEN_DINQ_ENABLE_LLM_REWRITE=true
```

## Evidence Constraints

The LLM can interpret intent, infer safe sources, plan generation, synthesize claims from evidence, and explain missing evidence. It cannot invent URLs, companies, papers, roles, skills, or evidence. Unsupported claims are discarded.

## Fallback Behavior

If LLM config is missing or invalid, OpenDinq returns `llmUsed: false` and uses deterministic fallback planning. Natural-language-only generation creates a manual profile seed and warns that stronger evidence requires a public source.

## Known Limitations

- LLM improves synthesis, but claims still require evidence.
- OpenDinq does not invent sources.
- Natural-language-only generation has weaker evidence unless sources are provided.
- No production web-wide entity search.
- No auth or ownership.
- No production semantic/vector search.
