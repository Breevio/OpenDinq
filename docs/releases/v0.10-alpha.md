# OpenDinq v0.10-alpha

v0.10-alpha makes profile generation search-first while preserving deterministic local mode.

## What Changed

- Added single-input search-first generation on `/generate`.
- Added candidate resolution with `POST /api/profiles/resolve`, `POST /api/profiles/search-and-generate`, and `POST /api/profiles/generate-from-candidate`.
- Added `POST /api/profiles/plan` for non-persistent plan previews.
- Added `POST /api/profiles/generate-ai` for plan-plus-generate flow.
- Added `@opendinq/llm` LLM-first planning with a unified `ProfileGenerationPlan`.
- Added MCP tools `opendinq_plan_profile_generation` and `opendinq_generate_profile_ai`.
- Moved explicit source fields under collapsed Advanced sources.
- Names, natural-language descriptions, and URLs are all valid inputs; URLs are optional shortcuts, not required.
- Natural-language input now attempts safe public source discovery before falling back to a reviewable workspace with user-provided claims and missing-evidence prompts.

## Env Vars

```bash
OPEN_DINQ_ENABLE_LLM_GENERATION=true
OPEN_DINQ_LLM_PROVIDER=openai-compatible
OPEN_DINQ_LLM_MODEL=gpt-4.1-mini
OPEN_DINQ_LLM_API_KEY=...
OPEN_DINQ_LLM_CHAT_COMPLETIONS_URL=https://api.openai.com/v1/chat/completions # optional
OPEN_DINQ_LLM_BASE_URL=https://api.openai.com/v1 # optional
OPEN_DINQ_LLM_TIMEOUT_MS=90000 # optional
OPEN_DINQ_LLM_MAX_TOKENS=1200 # optional
```

Card rewrite remains separately gated:

```bash
OPEN_DINQ_ENABLE_LLM_REWRITE=true
```

## Evidence Constraints

The LLM can interpret intent, suggest connector queries, rank confirmed candidates, convert user descriptions into user-provided claims, and explain missing evidence. It cannot invent URLs, companies, papers, roles, skills, candidates, or evidence. User-provided claims are not verified evidence.

## Fallback Behavior

If LLM config is missing, times out, or returns unusable JSON, OpenDinq returns `llmUsed: false` and uses local candidate search. If a connector fails, generation should still create or preserve a reviewable flow with warnings.

## Known Limitations

- LLM improves synthesis, but claims still require evidence.
- OpenDinq does not invent sources.
- Natural-language-only generation has weaker evidence unless candidates or public sources are found.
- Connector coverage is public-source only; LinkedIn/X/browser scraping remain out of scope.
- No production web-wide entity search.
- No auth or ownership.
- No production semantic/vector search.
